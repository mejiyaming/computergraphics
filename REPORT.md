## 1. 프로젝트 기획 개요 

### 1.1 프로젝트 목표
본 프로젝트는 컴퓨터 그래픽스 강의에서 학습한 씬 그래프, 카메라 좌표계 변환, 쿼터니언 회전, GI 개념 등을 안티그래비티 바이브 코딩을 통해 실제 웹 브라우저 환경에서 동작하는 3D 게임으로 구현해 보는 것을 목표로 삼았습니다.
게임은 해당 기술들을 적용하기 쉽도록 여러 가지 미니게임(퍼즐)을 푸는 방식으로 기획하여, 배운 개념들을 다양한 환경에서 다루어 보았습니다. 플레이어가 마법소녀 캐릭터를 조작하여 미로 탈출, 점프 챌린지, 색상 기억 퍼즐을 해결하고 세 가지 소울 젬(Pink, Blue, Gold)을 획득하여 마물 세계를 탈출하는 흐름으로 구성하였습니다.

### 1.2 프로젝트 컨셉 및 흐름
게임 컨셉은 마물 세계에 갇힌 마법소녀가 세 가지 포탈 속에서 각각 소울 젬을 획득하여 마물 세계를 탈출하는 내용으로 주제를 설정하였습니다. 각 방의 컨셉은 핑크, 블루, 골드로 나뉘며, 각각 미로 탈출을 통한 소울 젬 획득, 공중 발판 점프맵, 순서 기억 퍼즐로 구성하였습니다. 여러 가지 미니게임을 통해 다양한 조명 효과를 나타내보고 싶어 해당 주제로 정하게 되었습니다.

게임의 큰 흐름으로는 타이틀 화면에서 마법소녀 캐릭터의 이름을 설정하고 게임을 시작할 수 있으며, 이는 게임 동작 시 대화창 및 엔딩 크레딧 롤에도 반영되도록 하였습니다. 게임 시작 버튼을 누르면 대화 상자를 통해 게임에 대한 간략한 설명과 유저가 도달해야 할 최종 목표에 대한 정보를 전달합니다. 세부적인 지시사항은 화면에 안내 텍스트가 나타나도록 처리하여 게임 진행에 필요한 정보를 전달하였습니다.

게임의 공간은 크게 메인 광장인 Magical Hub, 핑크 포탈, 파란 포탈, 골드 포탈 세 가지로 구성되어 있습니다. 게임 시작 시 Magical Hub에 캐릭터가 생성되며, 이곳에서 각각의 미니게임을 위한 공간으로 포탈을 통해 입장할 수 있습니다. 포탈로 들어가 각각의 미니게임을 클리어하면 다시 Magical Hub로 돌아오며, 클리어된 포탈은 '클리어 완료'라고 표시되고 재입장이 불가능해집니다.

미니게임을 모두 클리어하여 세 가지 소울 젬을 획득하면 Magical Hub 중앙에 큰 무지개 및 포탈이 소환됩니다. 해당 포탈로 입장하면 마법소녀 캐릭터의 등에 날개가 달려 구름 위로 날아오르는 듯한 모션이 재생되고 화면 하단에서 크레딧이 올라오며, '타이틀 화면으로'라는 버튼을 통해 타이틀 화면으로 되돌아가 다시 플레이할 수 있습니다.

## 2. 강의 내용과의 매핑 및 그래픽스 기술 구현

### 2.1 씬 그래프 (Scene Graph) 계층 구조 관리
게임 내에서 미로 벽면, 점프 발판, 보석 등 여러 오브젝트가 실시간으로 생성되고 제거되다 보니, 전체 3D 월드 구성원들을 관리하기 위해 강의에서 학습한 씬 그래프 계층 구조를 사용했습니다. Three.js의 `Scene` 객체를 최상위 노드로 설정하고, 그 아래 플레이어 캐릭터, 맵 메쉬, 각종 조명 등을 자식 노드로 덧붙여 계층 구조 트리를 유지하며 오브젝트를 렌더링하고 업데이트하도록 구현했습니다.

### 2.2 카메라 좌표계 및 행렬 변환
플레이어를 따라다니는 3인칭 뷰 카메라를 구현하기 위해 카메라 좌표계의 이동 및 View 행렬 변환 공식을 사용했습니다. 플레이어의 위치를 기준 원점으로 정하고, 삼각함수를 써서 항상 캐릭터의 후방 일정 거리와 높이에 카메라가 위치하도록 좌표를 설정했습니다. 또한 카메라가 늘 캐릭터를 바라보도록 LookAt 행렬(View 행렬)을 매 프레임 업데이트했습니다.

여기에 덧붙여 플레이어 이동 시, 사용자의 입력 방향(2D 평면 벡터)을 그대로 사용하지 않고 카메라의 현재 시점 좌표계 기준의 3D 방향 벡터로 투영 변환하는 계산을 적용했습니다. 이를 통해 화면 상에서 카메라 시점 기준으로 키보드를 누르는 대로 직관적으로 캐릭터가 움직이게 만들었습니다.

### 2.3 쿼터니언을 활용한 회전 제어
캐릭터가 움직이는 방향으로 고개를 돌릴 때 오일러 각(Euler angles)을 사용하면 특정 축이 겹치는 짐벌 락(Gimbal Lock) 문제가 발생하거나 회전이 부자연스럽게 꼬였습니다. 이를 해결하기 위해 수업시간에 배운 쿼터니언(Quaternion)을 활용했습니다. 목표 방향의 회전 값을 계산한 후, Three.js의 `slerp`(구면 선형 보간)를 이용해 매 프레임 부드럽게 각도가 돌아가도록 구현했습니다.
  ```javascript
  // src/game/Player.js
  const targetRotation = new THREE.Quaternion();
  targetRotation.setFromAxisAngle(new THREE.Vector3(0, 1, 0), angle);
  this.mesh.quaternion.slerp(targetRotation, 0.15); // 프레임별 캐릭터 회전각 보간
  ```

## 3. Global Illumination (GI) 유사 효과 표현

과제 요구사항인 실시간 전역 조명(Global Illumination)의 시각적 느낌을 브라우저 환경에서 나타내기 위해, 실제 무거운 실시간 GI 알고리즘 연산을 사용하는 대신 다음과 같이 간접광 효과를 표현하였습니다.

### 3.1 간접광 느낌을 위한 보조 광원 배치
격자 형태로 프로브를 배치하여 간접 조도를 표현하는 DDGI 방식에서 아이디어를 얻어, 미로 코너나 발판 베이스 부분에 여러 개의 작은 `PointLight`들을 분산 배치했습니다. 빛이 벽이나 바닥에 부딪혀 튕겨 나가는 듯한 간접광 느낌을 내기 위해 배치한 보조 광원들입니다.

다만 미로 전체의 모든 꺾임점에 광원을 다 넣으면 프레임 드랍이 심해져서, 플레이어가 지나갈 때 눈에 띄는 주요 길목 위주로만 골라 배치하여 성능 저하를 방지했습니다.
  ```javascript
  // src/scenes/MazeScene.js
  const giLightPositions = [
    [this.startPos.x, 1.8, this.startPos.z],
    [OFFSET_X + COLS * CELL * 0.8, 1.8, OFFSET_Z + ROWS * CELL * 0.27],
    [OFFSET_X + COLS * CELL * 0.2, 1.8, OFFSET_Z + ROWS * CELL * 0.45],
    [OFFSET_X + COLS * CELL * 0.8, 1.8, OFFSET_Z + ROWS * CELL * 0.64],
    [OFFSET_X + COLS * CELL * 0.2, 1.8, OFFSET_Z + ROWS * CELL * 0.82],
  ];
  giLightPositions.forEach(([x, y, z]) => {
    const light = new THREE.PointLight(0xff1166, 2.0, 12, 2);
    light.position.set(x, y, z);
    s.add(light);
    this.objects.push(light);
  });
  ```

### 3.2 자체 발광 효과를 활용한 공간 표현
어두운 맵에서 벽 테두리와 퍼즐 제단 근처가 스스로 빛나는 느낌을 주기 위해 자체 발광(Emissive) 효과를 적용했습니다. 벽 테두리 몰딩 부분과 퍼즐 제단 바닥면에 자체 발광 옵션(`emissiveIntensity = 1.2`)을 넣은 얇은 상자 모양 메쉬들을 덧붙였습니다.

실제 실시간 전역 조명 연산을 돌리기에는 브라우저 환경에서 버벅임이 심했기 때문에, 발광하는 벽면 주변의 PointLight 조도 값을 살짝 높여 시각적으로 빛이 번져 나가는 것처럼 트릭을 주어 구현했습니다.
  ```javascript
  // src/scenes/MazeScene.js
  const trimMat = new THREE.MeshStandardMaterial({
    color:             0xff3388,
    emissive:          0xff1166, // 핑크색 자체 발광 설정
    emissiveIntensity: 1.2,
    roughness:         0.1,
  });
  ```

## 5. 결론 및 고찰

이번 학기 동안 컴퓨터 그래픽스 강의에서 이론으로 배웠던 여러 3D 그래픽스 기술들을 직접 코딩하며 웹 게임으로 만들어 볼 수 있었습니다. 추상적으로만 이해했던 변환 행렬이나 쿼터니언, 조명 배치 같은 개념들이 브라우저 화면에 실시간으로 반영되는 모습을 눈으로 확인하며 더 깊게 배울 수 있었습니다.

### 프로젝트를 하며 아쉬웠던 점 및 향후 계획
원래 기획했던 마법소녀 컨셉에 맞추어 더 다양하고 퀄리티 높은 3D 모델이나 UI 에셋을 사용하고 싶었으나, 조건에 맞는 무료 에셋을 구하는 데 한계가 있었습니다. 어쩔 수 없이 Three.js의 기본 도형 상자나 튜브 등을 결합하여 요술봉, 리본 등을 어설프게나마 직접 구현하여 사용했습니다. 나중에 Blender 같은 모델링 툴을 더 공부해서 더 예쁜 3D 에셋들을 리소스로 적용해보고 싶습니다.

### 최종 소감
씬 그래프, 카메라 오빗 제어, 쿼터니언 회전 등 개념 자체는 단순해 보였지만, 실제로 캐릭터가 점프하거나 포탈을 이동할 때 카메라 앵글이 갑자기 튀거나 캐릭터 회전이 꼬이는 등의 물리/수학적 예외 상황이 많이 발생했습니다. 이를 디버깅하는 과정에서 그래픽스 이론을 더 꼼꼼히 이해하게 되었고, 단순 수학 연산을 넘어 사용자의 관점에서 매끄러운 3D 인터랙션을 구현하는 데 많은 정교한 처리가 필요한다는 것을 깨달았습니다.
