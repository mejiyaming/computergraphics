## 1. 프로젝트 기획 개요 

### 1.1 프로젝트 목표
본 프로젝트는 컴퓨터 그래픽스 강의에서 학습한 씬 그래프, 카메라 좌표계 변환, 쿼터니언 회전, GI 개념 등을 안티그래비티 바이브 코딩을 통해 실제 웹 브라우저 환경에서 동작하는 3D 게임으로 구현해 보는 것을 목표로 삼았습니다.
게임은 해당 기술들을 적용하기 쉽도록 여러 가지 미니게임(퍼즐)을 푸는 방식으로 기획하여, 배운 개념들을 다양한 환경에서 다루어 보았습니다. 플레이어가 마법소녀 캐릭터를 조작하여 미로 탈출, 점프 챌린지, 색상 기억 퍼즐을 해결하고 세 가지 소울 젬(Pink, Blue, Gold)을 획득하여 마물 세계를 탈출하는 흐름으로 구성하였습니다.

### 1.2 프로젝트 컨셉 및 흐름
게임 컨셉은 마물 세계에 갇힌 마법소녀가 세 가지 포탈 속에서 각각 소울 젬을 획득하여 마물 세계를 탈출하는 내용으로 주제를 설정하였습니다. 각 방의 컨셉은 핑크, 블루, 골드로 나뉘며, 각각 미로 탈출을 통한 소울 젬 획득, 공중 발판 점프맵, 순서 기억 퍼즐로 구성하였습니다. 여러 가지 미니게임을 통해 다양한 조명 효과를 나타내보고 싶어 해당 주제로 정하게 되었습니다.

게임의 큰 흐름으로는 타이틀 화면에서 마법소녀 캐릭터의 이름을 설정하고 게임을 시작할 수 있으며, 이는 게임 동작 시 대화창 및 엔딩 크레딧 롤에도 반영되도록 하였습니다. 게임 시작 버튼을 누르면 대화 상자를 통해 게임에 대한 간략한 설명과 유저가 도달해야 할 최종 목표에 대한 정보를 전달합니다. 세부적인 지시사항은 화면에 안내 텍스트가 나타나도록 처리하여 게임 진행에 필요한 정보를 전달하였습니다.

게임의 공간은 크게 메인 광장인 Magical Hub, 핑크 포탈, 파란 포탈, 골드 포탈 세 가지로 구성되어 있습니다. 게임 시작 시 Magical Hub에 캐릭터가 생성되며, 이곳에서 각각의 미니게임을 위한 공간으로 포탈을 통해 입장할 수 있습니다. 포탈로 들어가 각각의 미니게임을 클리어하면 다시 Magical Hub로 돌아오며, 클리어된 포탈은 '클리어 완료'라고 표시되고 재입장이 불가능해집니다.

미니게임을 모두 클리어하여 세 가지 소울 젬을 획득하면 Magical Hub 중앙에 큰 무지개 및 포탈이 소환됩니다. 해당 포탈로 입장하면 마법소녀 캐릭터의 등에 날개가 달려 구름 위로 날아오르는 듯한 모션이 재생되고 화면 하단에서 크레딧이 올라오며, '타이틀 화면으로'라는 버튼을 통해 타이틀 화면으로 되돌아가 다시 플레이할 수 있습니다.

## 2. 강의 내용과의 매핑 및 그래픽스 기술 구현 (Lecture-to-Code Mapping)

### 2.1 씬 그래프 (Scene Graph) 계층 구조 관리
- **씬 트리를 통한 오브젝트 관리**: 게임이 진행되면서 미로 벽면, 플랫폼 발판, 보석 등 다양한 오브젝트가 실시간으로 생성되고 제거되기 때문에 Scene Graph 구조를 활용하여 관리하였습니다. Three.js의 `Scene` 객체를 최상위 루트 노드로 삼고, 하위에 각각의 메쉬 노드들을 계층적으로 추가하여 3D 월드를 구성하였습니다.

### 2.2 카메라 좌표계 및 행렬 변환 (Camera & Transforms)
- **3인칭 뷰 카메라 좌표계**: 플레이어를 추적하는 3인칭 오빗 카메라를 만들기 위해 카메라 좌표계와 View 행렬 변환을 활용하였습니다. 플레이어의 중심 좌표를 원점으로 삼아 삼각함수 연산식으로 카메라의 3차원 위치 좌표를 결정하고, 카메라가 플레이어를 응시하도록 View 행렬 변환을 실시간으로 업데이트해 주었습니다.
- **좌표계 전환**: 플레이어가 키보드로 입력하는 2차원 평면 방향 벡터를 3차원 카메라 시점의 월드 좌표계 방향으로 투영하여 캐릭터가 카메라가 보는 방향을 기준으로 자연스럽게 이동할 수 있도록 변환 처리를 하였습니다.

### 2.3 쿼터니언 (Quaternion)을 활용한 회전 제어
- **쿼터니언 회전 적용**: 캐릭터가 이동 방향에 맞춰 급격히 회전할 때 오일러 각을 사용할 경우 특정 축이 겹쳐 회전 연산이 꼬이는 짐벌 락 (Gimbal Lock) 현상이 발생하였습니다. 이를 해결하기 위해 수업시간에 배운 쿼터니언(Quaternion) 구면 선형 보간(Slerp)을 도입하여, 프레임별로 끊김 없고 매끄러운 3D 회전 처리가 이루어지도록 구현하였습니다.
  ```javascript
  // src/game/Player.js
  const targetRotation = new THREE.Quaternion();
  targetRotation.setFromAxisAngle(new THREE.Vector3(0, 1, 0), angle);
  this.mesh.quaternion.slerp(targetRotation, 0.15); // 프레임별 캐릭터 회전각 보간
  ```

## 3. Global Illumination (GI) 유사 효과 표현 (Core GI Sim)

과제 요구사항인 실시간 전역 조명(Global Illumination)의 시각적 느낌을 브라우저 환경에서 나타내기 위해, 실제 무거운 실시간 GI 알고리즘 연산을 사용하는 대신 다음과 같이 간접광 효과를 표현하였습니다.

### 3.1 간접광 느낌을 위한 보조 광원 배치
- **구현 방식**: 프로브를 격자 형태로 두고 간접 조도를 계산하는 DDGI의 아이디어를 참고하여, 미로 코너나 발판 베이스에 **가상 조도 탐침** 역할을 할 수 있는 여러 개의 작은 `PointLight`들을 분산 배치하였습니다. 이를 통해 실제로 빛이 튕겨 나간 것과 같은 간접광 분위기를 표현하였습니다.
- **성능 고려 사항**: 미로 안의 모든 꺾임점에 보조 광원을 배치하면 렌더링 부하가 증가하였습니다. 따라서 필요한 위치만 선별하여 배치함으로써 렌더링 부하를 줄였습니다.
  ```javascript
  // src/scenes/MazeScene.js
  const giLightPositions = [
    [this.startPos.x, 1.8, this.startPos.z],
    [OFFSET_X + COLS * CELL * 0.8, 1.8, OFFSET_Z + ROWS * CELL * 0.27],
    [OFFSET_X + COLS * CELL * 0.2, 1.8, OFFSET_Z + ROWS * CELL * 0.45],
    [OFFSET_X + COLS * CELL * 0.8, 1.8, OFFSET_Z + ROWS * CELL * 0.64],
    [OFFSET_X + COLS * CELL * 0.2, 1.8, OFFSET_Z + ROWS * CELL * 0.82],
  ];
  giLightPositions.forEach(([x, y, z]) => {
    const light = new THREE.PointLight(0xff1166, 2.0, 12, 2);
    light.position.set(x, y, z);
    s.add(light);
    this.objects.push(light);
  });
  ```

### 3.2 자체 발광 효과를 활용한 공간 표현
- **구현 방식**: 벽 테두리와 제단 주변에 빛이 퍼지는 느낌을 주기 위해 자체 발광 효과를 적용하였습니다. 벽 몰딩 테두리와 퍼즐 제단 베이스에 자체 발광 속성(`emissiveIntensity = 1.2`)을 지닌 얇은 상자 띠 형태의 메쉬들을 추가하였습니다.
- **광분산 처리**: 자체 발광 속성을 사용하여 주변 공간의 밝기 변화를 나타냈으며, 발광 벽면 주변이 조금 더 밝아 보이도록 처리하였습니다. 이는 실제 GI 연산이 아닌 시각적으로 빛이 번지는 모습을 나타내기 위한 방식을 적용한 것입니다.
  ```javascript
  // src/scenes/MazeScene.js
  const trimMat = new THREE.MeshStandardMaterial({
    color:             0xff3388,
    emissive:          0xff1166, // 핑크색 자체 발광 설정
    emissiveIntensity: 1.2,
    roughness:         0.1,
  });
  ```

## 5. 결론 및 고찰

이번 프로젝트는 컴퓨터 그래픽스 수업에서 배운 여러 3D 그래픽스 이론들을 직접 게임의 형태로 구현하며, 이론적인 개념들이 실제 화면에서 어떻게 제어되고 렌더링되는지 확인하는 과정이었습니다.

### 향후 개선하고 싶은 사항
- **그래픽 소재 및 리소스 수급**: 주제와 걸맞게 더 화려한 UI 및 인터페이스를 사용하여 각각의 아이템을 디자인하고 싶었지만, 마법소녀 컨셉트에 어울리는 무료 3D 에셋을 찾는 데 어려움이 있었습니다. 이에 따라 큐브와 기본 도형들을 결합하여 요술봉이나 리본 등 필요한 오브젝트를 직접 제작해 사용하였는데, 차후 에셋 제작 툴을 다루어 리소스를 보완하고 싶습니다.

### 강의 이론과 구현의 연결 및 종합 소감
강의에서 배운 씬 그래프, 카메라 좌표계, 쿼터니언, 전역 조명 등의 개념은 수업에서는 이론 위주로 학습하였으나, 직접 프로그램을 작성해 보면서 실제 화면에 어떤 영향을 주는지 확인할 수 있었습니다. 특히 카메라 제어와 쿼터니언 회전 구현 시 예상보다 많은 디버깅 과정이 필요하였으며, 그래픽스 이론뿐만 아니라 이를 구현하는 프로그래밍 역량이 필요하다는 점을 배웠습니다.
